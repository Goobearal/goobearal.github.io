CSS Selectors and Specificity

CSS (cascading style sheet) is how simple and plain html code is transformed into an appealing and attractive site. However, within CSS, it is important to understand the different selectors available to the programmer as well as their specificity in order to have a fluid coding experience.

Some common CSS selectors:
CSS Selectors What it is Example
* {} This, a universal selector, allows the CSS code and styling to apply to all of the basic html elements, which includes "p", "span", and "div" elements.
p {} or h1 {} This, a type selector, applies the CSS styling to elements in the html file. While "*" applies to all the elements, having just the type name will limit the styling to that element.
.class This, a class selector, applies CSS styles to html elements with a specific class. This allows different html elements to be styled in the same way in one fell swoop.
CSS specificity:
CSS Selector Types What it means Specificity
n {} Type selectors are the least specific CSS selector, with the universal selector being least specific. Both class and ID selector CSS styling will be applied over CSS styling by type selectors, since both are more specific.
.n Class selectors are a bit more specific than type selectors. These address certain classes, which allows the programmer to selectively style html elements. This is more specific than type selectors, but less specific than ID selectors. So its CSS stling will be applied instead of styling done with type selectors, but not over styling done with ID selectors.
#n ID selectors are the most specific of these three CSS selectors. IDs are specific to only one html element, so unlike a class, it won't be applied to multiple html elements if the programmer wants it to. This is the most specific, and therefore the CSS styling in this selector takes priority over the other two selectors.
